Storage

Default. Your node validates the chain but discards old blocks. ~30 GB disk.
Keep the last N days of blocks. Disk usage grows roughly with N (current chain rate ≈ 1 GB/day).
Keep all blocks since you enable archival. Disk usage starts at ~1.5 TB and grows ~10–20 GB/month. Switching to archival now does NOT recover history from before the switch — your node only archives forward from today. To get older history, import a snapshot from another archival operator after switching.
Archival mode is disabled on testnet because testnet resets can permanently delete archival data. Switch to mainnet to use archival.

Archive seed sources

wRPC URLs of archival kaspad nodes the fill loop can pull from when local kaspad has pruned a range. One per line. Every block pulled is hash-verified against your local kaspad's PoW chain — these sources don't need to be trusted, the math catches lies.
For the archival operator: run kaspad with --archival --rpc-listen=0.0.0.0:18110, then open port 18110 (or use a Tailscale/Cloudflare Tunnel for private sharing).

Get help

When something breaks, MyKAI sends a technical report so we can help — often without you having to do anything. Personal info is redacted.
Report a problem
If your node has a specific issue you want to tell us about, copy a report and paste it into our Discord or email.
Save your progress
Your blocks, uptime, and other achievements live with this node. MyKAI keeps a backup file in your Documents folder so you don't lose them if your computer breaks or you reinstall. For extra safety, copy that file somewhere else — OneDrive, a USB stick, or wherever you keep important things.
Chain data
If your sync is stuck for hours and nothing else helps, this deletes the local Kaspa chain database (~37 GB) and starts a fresh sync. Takes 2–4 hours but reliably finishes. Your accountKey and stats are preserved.
Stop the node
Shuts down kaspad and disconnects from the network. The app stays open so you can start it again later. Use this if you need to free up CPU or bandwidth, or if you're troubleshooting an issue.
